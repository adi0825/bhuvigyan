## SECTION 1 — DATABASE SCHEMA (PostgreSQL + PostGIS)

### Rules
- All tables use UUID primary keys (`uuid DEFAULT gen_random_uuid()`).
- All tables have `created_at TIMESTAMPTZ DEFAULT now()` and `updated_at TIMESTAMPTZ DEFAULT now()`.
- Foreign keys are always explicit with `ON DELETE RESTRICT` unless stated otherwise.
- Enum types are declared as PostgreSQL `CREATE TYPE`.
- All geometry columns use `GEOGRAPHY(POLYGON, 4326)` or `GEOGRAPHY(POINT, 4326)`.
- Indexes are declared inline below each table.

## SECTION 3 — API CONTRACTS

Base URL: `/api/v1`  
Auth header: `Authorization: Bearer {jwt}`  
All responses: `Content-Type: application/json`  
Error envelope: `{ "error": { "code": "STRING", "message": "string", "traceId": "string" } }`  
Success envelope: `{ "data": {}, "meta": { "traceId": "string", "timestamp": "ISO8601" } }`

---

### 3.1 Auth

```
POST   /auth/otp/send         { mobile }                        → { otpToken }
POST   /auth/otp/verify        { otpToken, otp }                 → { accessToken, refreshToken, user }
POST   /auth/login             { mobile, password }              → { accessToken, refreshToken, user }
POST   /auth/refresh           { refreshToken }                  → { accessToken }
POST   /auth/logout            (header)                          → 204
```

## SECTION 11 — ADDITIONAL MISSING CONTRACTS (beyond original checklist)

### 11.1 Claim Number Generation
Format: `BHU-{STATECD}-{YYYYMM}-{6-digit-zero-padded-sequence}`. Example: `BHU-MH-202411-000042`. Sequence is per-state per-month using a PostgreSQL sequence: `claim_seq_{state}_{year}{month}`.

### 11.2 Trace ID Generation
Format: UUID v4 without hyphens, 32 chars. Generated at API gateway on request entry and propagated in all downstream calls via `X-Trace-Id` header.

### 11.3 Signed URL Contract
MinIO (or S3-compatible) pre-signed URL TTL: 15 minutes for evidence PDF downloads. 1 hour for satellite image downloads. Signed URL generation must be logged in audit with actor and target file key.

### 11.4 Outbox Poller Schedule
APScheduler or Spring `@Scheduled`: poll every 30 seconds. Max batch: 50 events per run. Retry max: 5 attempts with exponential backoff (30s, 60s, 120s, 300s, 600s). After 5 failures: mark `FAILED`, publish `fraud.alert` with `alertType: OUTBOX_DEAD_LETTER`.

### 11.5 Model Versioning Contract
All Python ML models stored in MinIO under `models/{modelName}/{version}/model.pkl`. Model metadata in PostgreSQL table `model_registry (id, model_name, version, accuracy, trained_at, is_active, feature_version)`. Scoring service loads active model at startup and hot-reloads on `model.updated` Kafka event.

### 11.6 NDVI Interpretation Thresholds (rule inputs)
```
NDVI < 0.10  → barren / no crop (flag: barren_pre_sowing)
0.10–0.30    → stressed or sparse crop
0.30–0.60    → moderate vegetation
0.60–0.80    → healthy crop
> 0.80       → dense / very healthy crop
```
If claimed_loss_pct > 60 AND ndvi_mean_claim > 0.45 → `ndvi_loss_contradiction = TRUE` → hard weight 0.31 in ensemble.

### 11.7 Docker Compose Services List
Services: `postgres`, `redis`, `kafka`, `zookeeper`, `cassandra`, `minio`, `api-gateway`, `auth-service`, `farmer-service`, `claims-service`, `land-adapter-service`, `satellite-service`, `feature-service`, `scoring-service`, `evidence-service`, `notification-service`, `field-visit-service`, `admin-service`, `audit-service`, `analytics-service`, `scheduler-service`, `farmer-portal`, `csc-portal`, `officer-portal`, `insurer-portal`, `admin-portal`. All services: `restart: unless-stopped`, `healthcheck` defined, `depends_on` with `condition: service_healthy`.

### 11.8 Environment Variables (minimum required per service)
Every service must read from environment:
- `SPRING_DATASOURCE_URL` / `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USER` / `DB_PASS`
- `REDIS_HOST` / `REDIS_PORT`
- `KAFKA_BOOTSTRAP_SERVERS` (optional, fall back if blank)
- `MINIO_ENDPOINT` / `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` / `MINIO_BUCKET`
- `JWT_SECRET` / `JWT_ACCESS_TTL_MINUTES` / `JWT_REFRESH_TTL_DAYS`
- `APP_ENV` (dev/test/stage/prod)
- `STATE_ADAPTER_{MH|KA|RJ|TG|PB|UP}_URL`
- `OPENAI_API_KEY` (for GPT-3.5 threat narration — optional, disabled in dev)
